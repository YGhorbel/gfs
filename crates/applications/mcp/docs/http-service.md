# Running the MCP server (rmcp 0.16)

This crate uses **rmcp 0.16** with **stdio** (default) or **streamable HTTP**.

## Handler pattern

- **Handler struct:** `GfsMcpHandler` uses rmcp 0.16's macro-based API (same as mcp2):
  - `#[tool_router]` + `#[tool(description = "...")]` on each tool method — tools are declared with request structs (`Parameters<T>`) and `schemars::JsonSchema` for input schemas.
  - `#[tool_handler] impl ServerHandler` — provides only `get_info()`; `list_tools` and `call_tool` are generated by the macros.
  - `ToolRouter<Self>` field — initialized with `Self::tool_router()` in `new()`.
- **Tools:** Seven tools (list_providers, status, commit, log, checkout, init, compute). Each is an async `#[tool]` method taking `Parameters<RequestStruct>` and returning `Result<CallToolResult, McpError>`.

## Transports

- **Stdio:** `gfs-mcp --stdio` or when run with no args **and** stdin is not a TTY (e.g. Cursor spawns the binary). Uses stdin/stdout for JSON-RPC.
- **Streamable HTTP:** When you run `gfs-mcp` with no args in a terminal (TTY), it defaults to **HTTP on port 3000**. Or run `gfs-mcp --http [PORT]` explicitly. Then:
  - **POST** `http://127.0.0.1:PORT/mcp` — JSON-RPC. The server returns `Mcp-Session-Id` in the response header for session-based communication.
  - Clients send messages via HTTP POST; servers respond with JSON or SSE streams as per the streamable HTTP transport.

**Daemon:** `gfs mcp start` spawns the binary with `--http`.

## No authentication

**gfs-mcp does not require authentication.** There are no API keys, tokens, or login. The HTTP transport uses a **session ID** only to associate requests; that is not auth. If you see a message like *"Unauthorized"* or *"Authentication required"*, that comes from your **client**, not from gfs-mcp. Connect directly to `http://127.0.0.1:PORT/mcp`; no token or auth is required by this server.

## MCP Inspector compatibility

- **Protocol version:** The server advertises the default protocol version (rmcp 0.16) so clients that validate with Zod accept the response.
